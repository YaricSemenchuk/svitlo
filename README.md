# Svitlo — таксі як PWA

Єдиний застосунок таксі **Svitlo** на React (Vite). Дві ролі — **пасажир** і **водій** —
в одному кодбейсі, ставиться на домашній екран iOS/Android з одного URL, без App Store /
Google Play. Ринок — Україна, інтерфейс українською, валюта ₴.

## Запуск

```bash
npm install
npm run dev      # розробка
npm run build    # продакшн-збірка → dist/
npm run preview  # локальний прев'ю продакшн-збірки
```

Іконки PWA генеруються скриптом (вже згенеровані в `public/icons/`):

```bash
node scripts/gen-icons.mjs
```

## Флоу

**Пасажир:** `Order → Matching → (matched) → EnRoute → (arrived) → Trip → Complete → Order`
**Водій:** `Online → (request) → Request → (accept) → Navigate → Trip → Online`

Переходи `matched / request / arrived` приходять із `src/lib/realtime.js`, а не зашиті в екрани.

## Реєстрація та профілі

При першому виборі ролі — **реєстрація**:
- **Замовник:** ім’я, телефон, спосіб оплати (Apple/Google Pay, картка Mono, готівка).
- **Водій:** ім’я, телефон, марка/модель авто, колір, **номер машини**.

Профілі зберігаються в `localStorage` (`svitlo.profiles`) і переживають перезавантаження;
на головному екрані видно збережений профіль із кнопкою «змінити».

**Дані видно іншій стороні.** Зареєстрований водій відгукується пасажиру першим оффером —
з реальним авто, номером і телефоном; у поїздці пасажир бачить ці дані й може зателефонувати
(`tel:`). Профіль замовника (ім’я, оплата, телефон) показується водієві у вхідному запиті.

## «Назви свою ціну»

На екрані замовлення пасажир сам вписує **Звідки/Куди** і пропонує **свою ціну**
(степпер ±10 ₴ або ручний ввід; підказка показує відхилення від рекомендованої).

**Аукціон офферів.** Після `Запропонувати` екран `Matching` показує живий список
відгуків водіїв: хтось згоден **за вашою ціною**, хтось дає **контрпропозицію** (+₴).
Кожен оффер — водій з ★рейтингом, авто, ETA та дистанцією подачі. Пасажир обирає
будь-який — узгоджена ціна стає тарифом поїздки (лічильник і чек). Оффери приходять
із `realtime.js` (подія `ride:offer`, команда `ride:select`).

Пропозиція пасажира (адреси + ціна) також їде водієві у вхідному запиті як
**ОФЕР ПАСАЖИРА** (роль водія).

## Архітектура

- `src/lib/realtime.js` — **WebSocket-заглушка** з інтерфейсом справжнього сокета
  (`connect/disconnect/on/emit`). Уся realtime-логіка лише тут. Щоб під'єднати бойовий
  бекенд — заміни нутрощі файлу на `WebSocket` / `socket.io-client`, екрани не чіпати.
- `src/state/TripContext.jsx` — стейт-машина поїздки (`useReducer`) + місток realtime → dispatch.
  Стани: `idle → requesting → matching → assigned → arriving → arrived → in_trip → completed`.
- `src/lib/maps.js` — геометрія (`[lng,lat]`), маршрут по дорогах через **OSRM**, темний стиль
  карти (CARTO Dark Matter, без ключа).
- `src/components/LiveMap.jsx` — MapLibre GL: маршрут, маркери, demo-рух авто та режим навігатора
  (follow-камера з нахилом/поворотом для водія).
- `src/lib/useDriverTracking.js` — Screen Wake Lock + `geolocation.watchPosition` для водія.

## Деплой на статик-хостинг

`npm run build` → каталог `dist/` можна віддати на будь-який статичний хостинг.

- **Vercel:** `vercel --prod` (фреймворк Vite визначається автоматично).
- **Netlify:** publish directory `dist`, build command `npm run build`. Додай SPA-редірект:
  `/* /index.html 200` (файл `public/_redirects`).
- **Cloudflare Pages:** build command `npm run build`, output `dist`.

Усі SPA-маршрути (`/rider/*`, `/driver/*`) потребують fallback на `index.html` — service worker
це вже робить для офлайну, але для серверних 404 додай SPA-редірект на хостингу (як вище).

**Роздача по QR/посиланню:** на головному екрані є блок `InstallQR` — QR з `window.location.origin`
і кнопка встановлення. Підтримується атрибуція водія через `?ref=<id>` у посиланні з QR.

## Чесна примітка про обмеження PWA (водій)

Пасажир працює у foreground — обмежень нема. Водій тримає застосунок відкритим: увімкнено
**Screen Wake Lock** (екран не гасне) і трекінг через `watchPosition`. **При заблокованому
екрані iOS фоновий трекінг паузиться** — це обмеження PWA. Для MVP прийнятно; далі водія можна
винести в нативну обгортку (Capacitor), переиспользуючи цей фронт.

## Наступні кроки

- Реальний WebSocket-бекенд (заміна нутрощів `realtime.js`).
- Заміна OSRM на Google Directions / Mapbox (сигнатура `fetchRoute` та сама).
- Пуш-сповіщення (Web Push, iOS 16.4+).
- Нативна обгортка водія (Capacitor) для фонового трекінгу.
